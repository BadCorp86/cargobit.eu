# CargoBit Engineering Onboarding Guide
Version 1.0
Internal Use Only

---

# 1. Purpose

Dieses Dokument beschreibt den Onboarding-Prozess für neue Engineering-Mitarbeiter. Ziel ist es, neue Entwickler innerhalb von 48 Stunden produktiv zu machen.

---

# 3. Day 1

## 3.1 Morning (09:00 - 12:00)

| Time | Activity | Owner |
|------|----------|-------|
| 09:00 | Welcome & team intro | Manager |
| 09:30 | Equipment setup | IT Support |
| 10:00 | Account access verification | IT Support |
| 10:30 | Security training | Security Team |
| 11:30 | Tooling overview | Buddy |

## 3.2 Afternoon (13:00 - 17:00)

| Time | Activity | Owner |
|------|----------|-------|
| 13:00 | Architecture overview session | Tech Lead |
| 14:00 | Clone repositories | Self |
| 14:30 | Local development setup | Self + Buddy |
| 15:30 | Read Developer Handbook | Self |
| 16:30 | Pipeline overview | Buddy |

## 3.3 Day 1 Checklist

- [ ] All accounts created and working
- [ ] Equipment set up
- [ ] Slack channels joined
- [ ] Repos cloned
- [ ] Local environment running
- [ ] Developer Handbook read
- [ ] Architecture Overview understood

---

# 4. Day 2

## 4.1 Morning (09:00 - 12:00)

| Time | Activity | Owner |
|------|----------|-------|
| 09:00 | Multi-Agent System deep dive | Senior Engineer |
| 10:00 | Run tests locally | Self |
| 10:30 | Test exploration | Self |
| 11:30 | Code walkthrough | Buddy |

## 4.2 Afternoon (13:00 - 17:00)

| Time | Activity | Owner |
|------|----------|-------|
| 13:00 | Pick first ticket | Manager |
| 13:30 | Work on first ticket | Self |
| 15:00 | Code review process | Buddy |
| 15:30 | Continue ticket work | Self |
| 16:30 | Daily standup attendance | Self |

## 4.3 Day 2 Checklist

- [ ] Tests passing locally
- [ ] Multi-Agent System understood
- [ ] First ticket assigned
- [ ] Made first commit
- [ ] Attended standup

---

# 5. Week 1

| Day | Focus |
|-----|-------|
| Day 3-4 | Complete first ticket, first PR |
| Day 5 | PR merged, pick second ticket |

## 5.1 Week 1 Goals

- [ ] First PR merged
- [ ] All team members met
- [ ] Key processes understood
- [ ] Documentation contributed

---

# 6. Week 2-4

| Week | Focus |
|------|-------|
| Week 2 | Complete 2-3 tickets, deeper codebase understanding |
| Week 3 | Take on slightly larger ticket |
| Week 4 | Contribute to documentation or improve process |

## 6.1 Month 1 Goals

- [ ] 5+ PRs merged
- [ ] Participate in on-call shadow
- [ ] Complete security training
- [ ] Contribute to team documentation

---

# 7. 30-Day Check-in

## 7.1 Topics to Discuss

- How is the onboarding experience?
- What's working well?
- What could be improved?
- Any blockers or concerns?
- Goals for next 30 days

## 7.2 Feedback Collection

- Manager feedback
- Buddy feedback
- Peer feedback (optional)

---

# 8. Buddy Program

## 8.1 Buddy Responsibilities

- Daily check-ins (first week)
- Answer questions
- Guide through processes
- Introduce to team members
- Help with first PR

## 8.2 Buddy Assignment

| New Hire Level | Buddy Level |
|----------------|-------------|
| Junior | Senior+ |
| Mid-level | Senior+ |
| Senior | Staff+ |
