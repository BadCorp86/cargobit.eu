## J.1 Ziel & Scope

### Primary Goals

```
┌─────────────────────────────────────────────────────────────────────────────────────┐
│                          CONFIG-EDITOR OBJECTIVES                                    │
├─────────────────────────────────────────────────────────────────────────────────────┤
│                                                                                      │
│  1️⃣  RBAC/ABAC REGELN VERWALTEN                                                     │
│     • Rollen-Berechtigungen definieren                                              │
│     • Attribut-basierte Policies erstellen                                          │
│     • Änderungen mit Audit-Trail nachverfolgbar                                     │
│                                                                                      │
│  2️⃣  FRAUD-CONFIG EDITIEREN                                                         │
│     • Scoring-Parameter konfigurieren                                               │
│     • Weight-Verteilungen definieren                                                │
│     • Thresholds anpassen                                                           │
│                                                                                      │
│  3️⃣  RATE LIMITS KONFIGURIEREN                                                      │
│     • Endpoint-spezifische Limits                                                   │
│     • User-Group Limits                                                             │
│     • Burst-Konfiguration                                                           │
│                                                                                      │
│  4️⃣  VERSIONEN VERGLEICHEN                                                          │
│     • Diff-View zwischen Versionen                                                  │
│     • Rollback auf vorherige Version                                                │
│     • Change-History einsehen                                                       │
│                                                                                      │
│  5️⃣  VALIDATION IN ECHTZEIT                                                        │
│     • JSON-Schema Validation                                                        │
│     • Cross-Field Validation                                                        │
│     • Business Rule Checks                                                          │
│                                                                                      │
│  6️⃣  SAFE ROLLOUT                                                                  │
│     • Preview vor Apply                                                             │
│     • Dry-Run Modus                                                                 │
│     • Approvals für kritische Änderungen                                            │
│                                                                                      │
└─────────────────────────────────────────────────────────────────────────────────────┘
```

### Target Users

| User Role | Primary Tasks | Access Level |
|-----------|---------------|--------------|
| **Security Engineer** | Fraud-Config, AuthZ Rules | Full Access |
| **Platform SRE** | Rate Limits, Service Config | Full Access |
| **Backend Developer** | View Config, Debug | Read + Propose |
| **Compliance Officer** | Audit Trail, Review | Read Only |
| **Admin** | Approvals, User Mgmt | Full Access |

WebSocket_Events:

  config_updated:
    description: "Config version updated"
    payload:
      version: string
      changed_by: string
      timestamp: string
  
  validation_error:
    description: "Real-time validation error"
    payload:
      path: string
      message: string
      severity: "error" | "warning"
  
  approval_required:
    description: "Approval needed for change"
    payload:
      change_id: string
      approvers: string[]
      deadline: string
  
  approval_received:
    description: "Approval received"
    payload:
      change_id: string
      approver: string
      remaining: number
  
  # Client → Server
  subscribe:
    description: "Subscribe to config updates"
    payload:
      sections: string[]  # ["fraud", "rbac", "rate_limits"]
  
  preview_change:
    description: "Request preview of potential change"
    payload:
      config: object
      dry_run: boolean
